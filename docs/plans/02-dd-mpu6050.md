# Plan 02 — dd_mpu6050 (IMU device driver)

- **Epic:** [E03](../../.claude/memory/epics/E03-imu-driver.md)
- **Goal:** Implement the thin I2C wrapper for the MPU6050 that initialises the bus, verifies the chip, configures sensor ranges and DLPF, and returns one `imu_sample_t` per call in SI units.
- **Preconditions:** Plan 01 (scaffolding) has created `air-glove/lib/dd_mpu6050/` with `library.json` and `include/`. E02 has defined `ag_types.h` (`ag_result_t`, `imu_sample_t`, `AG_OK`, `AG_ERR_IO`, `AG_ERR_INIT`).

## Step-by-step

1. In `mpu6050_regs.h`, define register addresses:
   - `MPU6050_ADDR        0x68`
   - `REG_PWR_MGMT_1      0x6B`
   - `REG_CONFIG          0x1A`  (DLPF_CFG in bits [2:0])
   - `REG_GYRO_CONFIG     0x1B`  (FS_SEL in bits [4:3])
   - `REG_ACCEL_CONFIG    0x1C`  (AFS_SEL in bits [4:3])
   - `REG_ACCEL_XOUT_H    0x3B`  (start of 14-byte burst)
   - `REG_WHO_AM_I        0x75`
2. In `dd_mpu6050_init()`:
   1. `Wire.begin(21, 22); Wire.setClock(400000);`
   2. Read `REG_WHO_AM_I`; if != `0x68`, return `AG_ERR_IO`.
   3. Write `REG_PWR_MGMT_1 = 0x00` to clear sleep bit and select internal 8 MHz oscillator. Delay 2 ms.
   4. Write `REG_CONFIG = 0x03` (DLPF_CFG = 3, gyro/accel bandwidth ~44 Hz, 1 kHz gyro rate).
   5. Write `REG_GYRO_CONFIG = 0x08` (FS_SEL = 1 → ±500 °/s → LSB sensitivity 65.5 LSB/°/s).
   6. Write `REG_ACCEL_CONFIG = 0x08` (AFS_SEL = 1 → ±4 g → LSB sensitivity 8192 LSB/g).
   7. Store scale factors in file-scope `static` floats. Return `AG_OK`.
3. In `dd_mpu6050_read(imu_sample_t *out)`:
   1. If `out == nullptr`, return `AG_ERR_ARG`.
   2. Start burst read of 14 bytes at `REG_ACCEL_XOUT_H` via `Wire.beginTransmission(0x68); Wire.write(0x3B); Wire.endTransmission(false); Wire.requestFrom(0x68, 14)`.
   3. Byte-swap each int16 (MPU is big-endian; ESP32 is little-endian): `int16 = (hi << 8) | lo`.
   4. Skip bytes 6-7 (temperature) — not used in Phase I.
   5. Convert: `out->ax = ax_raw / 8192.0f * 9.80665f;` (etc. for ay, az). `out->gx = gx_raw / 65.5f * (float)(M_PI / 180.0);` (etc.).
   6. `out->t_us = (uint64_t) esp_timer_get_time();`
   7. If `Wire.endTransmission()` returned non-zero or `requestFrom` got < 14 bytes, return `AG_ERR_IO`.
4. Keep all state file-scope `static` inside `dd_mpu6050.cpp`. No global symbols beyond the two public functions.

## Public header (target signature)

```c
// lib/dd_mpu6050/include/dd_mpu6050.h
#pragma once
#include "ag_types.h"

// Initialise I2C bus on GPIO21 SDA / GPIO22 SCL at 400 kHz, verify WHO_AM_I,
// set +/-4 g, +/-500 deg/s, DLPF_CFG=3. Returns AG_OK on success, AG_ERR_IO
// when the chip does not respond. Thread-safety: call once from one task.
ag_result_t dd_mpu6050_init(void);

// Burst-read accel+gyro. Fills *out with m/s^2, rad/s, and esp_timer_get_time()
// timestamp in microseconds. Returns AG_OK or AG_ERR_IO on bus error.
// Thread-safety: caller must ensure single-producer access.
ag_result_t dd_mpu6050_read(imu_sample_t *out);
```

## Internal design notes

- `Wire.h` / `Arduino.h` are permitted because this is a `dd_*` library (see ADR-005). The alternative would be `driver/i2c.h` from ESP-IDF; staying on Wire simplifies the scaffold for Phase I and is explicitly allowed by the glossary.
- Single-register reads would cost 6 separate I2C transactions per sample; a 14-byte burst at 400 kHz completes in ~350 µs, comfortably under the 500 µs / `dd_mpu6050_read()` budget.
- DLPF_CFG = 3 gives ~44 Hz accel and ~42 Hz gyro bandwidth with 4.9 ms / 4.8 ms group delay — matched pair so Madgwick's accel-vs-gyro fusion is not phase-skewed.
- `±4 g` range handles hand-swing peaks (~2 g typical) without clipping yet keeps ~4 mg resolution. `±500 °/s` covers normal wrist rotation (~300 °/s peak) with 0.015 °/s/LSB.
- No FIFO, no DMA, no data-ready IRQ in Phase I. At 100 Hz the polled cost is low; IRQ mode is an E14 optimisation.
- State layout (file-scope static): `accel_scale_mps2`, `gyro_scale_rads`, `initialized`. Total ~16 bytes.

## Verification

On-target Unity test (`test/test_dd_mpu6050/test_main.cpp`), runs under `env:esp32dev`:

- `test_who_am_i_is_0x68` — raw I2C read of register 0x75 equals `0x68`.
- `test_init_returns_ok_on_connected_board` — `dd_mpu6050_init()` returns `AG_OK`.
- `test_read_returns_ok_and_populates_sample` — call fills `out->t_us != 0`.
- `test_gravity_magnitude_at_rest` — with the board flat on the bench, compute `|a| = sqrt(ax^2+ay^2+az^2)` and `TEST_ASSERT_TRUE(|a| >= 9.5f && |a| <= 10.1f)`.
- `test_gyro_bias_at_rest` — `TEST_ASSERT_TRUE(fabsf(gx) < 0.05f && fabsf(gy) < 0.05f && fabsf(gz) < 0.05f)`.
- `test_read_latency_under_500us` — surround `dd_mpu6050_read()` with `esp_timer_get_time()` bracket; assert `< 500`.

Manual bring-up probes:

- `printf("whoami=0x%02X\n", who);` at the end of init — must print `0x68`.
- Logic-analyser check: 400 kHz SCL (2.5 µs period) with START / address-byte `0xD0` (read bit high) after address write.

PlatformIO command:

```
pio test -e esp32dev -f test_dd_mpu6050
```

## Rollback / risk

- Smallest revert: delete `src/dd_mpu6050.cpp` body and replace with `return AG_OK;` stubs; the rest of the firmware still builds because `srv_*` consumers only depend on the header signatures.
- Known risks:
  - Missing I2C pull-ups on custom boards (breakout includes them; bare modules do not). Early warning: `WHO_AM_I` reads `0xFF` or bus NACK.
  - Address conflict with a second 0x68 device (e.g., DS1307 RTC). Not present on the AirGlove BOM; flag if anyone adds one.
  - Gyro scale error of 2× is invisible on a static bench — watch the fusion convergence rate during plan 05 bring-up.
